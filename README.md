## 📖 核心理念

### 道：守一而止
- 一次只写一章
- 一章只有一个核心目标
- 不跳写、不并行、不贪多

### 法：实事求是
- 朗读验证：写完必须朗读
- 数据验证：时间、数字必须复查
- 人设验证：言行必须符合设定

### 术：肌肉记忆
- 严格按照11步流程执行
- 重复形成本能反应
- 降低认知负担

### 器：工具辅助
- edge-tts：配音验证
- ffmpeg：视频预览
- git：版本控制

## 🔧 11步写作闭环

```
1. 朗读原文 → 2. 标注扩写点 → 3. 人物心理扩展
    ↓
4. 场景氛围扩展 ← 5. 对话艺术扩展 ← 6. 战斗艺术扩展
    ↓
7. 第N次朗读验证 → 8. 视频预览生成 → 9. 交叉验证
    ↓
10. 第N+1次朗读定稿 → 11. 记录到扩写统计表
```

## 📁 文件结构

```

├── outline.md            # 详细大纲（每章500字+）
├── characters/           # 人物设定
│   ├── protagonist.md
│   └── supporting/
├── world/               # 世界观设定
│   ├── timeline.md
│   └── map.md
├── chapters/            # 章节文件
│   ├── chapter_001.md
│   └── ...
├── stats.md             # 扩写统计表
└── review/              # 审阅记录
```
